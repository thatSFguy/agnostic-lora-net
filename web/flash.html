What are you flashing?

Pick the board, then connect it over USB. New boards arrive in their UF2 bootloader; a board already running our firmware is rebooted into it automatically.

Experimental boards aren’t published to the web flasher yet — build & flash them locally: pio run -e tracker_t1000_e -t upload (T1000-E / LR1110) or -e heltec_v4 (Heltec V4 / ESP32, via esptool).

Firmware is served from ./fw/ (same origin) — leave this blank to use it. You can paste the GitHub release URL as a fallback (UF2 download only; serial DFU may be blocked by cross-origin rules).

Provision & configure

Connect to the node, then everything below acts on it. USB (Web Serial) configures any node, including BLE security. BLE (Web Bluetooth) works on an already-paired node with BLE enabled — BLE can’t change its own security, so enable/PIN changes still need USB.

Battery calibration

reading: —

Measure the battery with a meter, enter the value, then Calibrate. The node derives its ADC scale from that one reading, stores it in flash (survives firmware updates), and reports battery as mV + % from then on.

Mobility

this node is: —

Flag a node that moves around. The controller gives it a higher margin reserve and an asymmetric loop — raises power fast when the link weakens, trims slowly — versus a fixed node it optimises straight down. Stored in flash and reported in telemetry.
